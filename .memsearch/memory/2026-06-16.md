- Claude Code concluded the bottleneck at thousands of units is simulation performance, not the Phaser engine itself, and that the same architectural fixes would be required in any engine.
- Claude Code identified four engine-agnostic optimizations: spatial hash grid for neighbor queries, AI time-slicing/LOD for off-screen units, fixed-timestep sim loop decoupled from render, and RenderTexture caching for procedural sprites.
- Claude Code noted that Godot's structural advantage (true multithreading + compiled hot paths) only matters after proving the JS thread is genuinely saturated, and recommended doing architecture work in Phaser first before considering migration.
